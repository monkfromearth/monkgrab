Why it exists

Browsers deliberately block one site from reading an iframe served by another domain. So embedded readers, course lessons, and docs viewers come out empty when you try to "save as Markdown." Copy-paste and bookmarklets hit the same wall.

MonkGrab sidesteps it the only way Chrome allows: it injects its converter into every frame of the page, so the code runs inside each frame's own origin where the cross-origin wall doesn't apply. It reads each frame's DOM, turns it into Markdown on the spot, keeps the main content, drops the ad and widget frames, and saves one .md file.

What you get

One click or one hotkey

⌘⇧G on Mac, Ctrl⇧G elsewhere. No popup, nothing to learn.

The content every other tool drops. The whole reason MonkGrab exists.

Install

  1. Download or clone the repository.
  2. Open Chrome at chrome://extensions.
  3. Toggle Developer mode ON (top-right).
  4. Click Load unpacked and select the folder with manifest.json.
  5. Pin the icon. Optionally set the hotkey at chrome://extensions/shortcuts.

Works on Chrome 102+ and Chromium browsers with Manifest V3 (Edge, Brave, Arc, Opera).

Use it

Open any page, then click the MonkGrab icon or press the hotkey. A teal ✓ flashes on success (a red ! if something went wrong). The file lands at:

Downloads/monkgrab/<domain>_<slug>_<date>_<time>.md

Permissions & privacy

MonkGrab collects nothing and sends nothing anywhere. No analytics, no telemetry, no accounts, zero network requests. The Markdown is written straight to your Downloads folder and nowhere else. Full policy on GitHub.

How it works

One Manifest V3 service worker. No content scripts, no popup, no remote code, no build step.

  1. A click or hotkey calls grab().
  2. It injects extractFrame into every frame via chrome.scripting.executeScript({ allFrames: true }) — each runs in its own origin, defeating the cross-origin wall.
  3. Each frame walks its DOM from the best content root (main → article → body) into Markdown, resolving relative URLs to absolute.
  4. Frames are ranked by text length; the main one is kept and only substantial extra frames are appended (ads/widgets filtered out).
  5. The Markdown is downloaded as a data: URL — the right approach since createObjectURL is unavailable in MV3 workers.

It's one heavily-commented file: background.js.
